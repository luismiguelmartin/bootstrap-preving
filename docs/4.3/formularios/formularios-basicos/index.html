        Formularios de Bootstrap

        Boostrap contiene multitud de componentes y disposiciones de elementos de formulario que nos permiten elaborar interfaces de forma rápida y con una base de desarrollo común.

        No compartiremos tu correo con nadie.

        Elementos de formulario

        Campos de texto

        Los elementos <input>, <select> y <textarea> se estilan mediante la clase .form-control. Esta clase proporciona la apariencia general, cambios de estado, tamaño y otros…

        Desplegables, selectores y archivos

        Sin embargo, los elementos <input type="file">, selectores de archivos, se estilan mediante la clase .form-control-file. En el ejemplo que sigue se muestra junto con varios elementos.

        Campos en solo lectura, deshabilitados o en texto-plano

        Añadiendo el atributo readonly a un campo de formulario podemos evitar que se modifique el valor del campo. Los campos en modo solo lectura aparecen sombreados (como los campos deshabilitados) pero permiten retener el foco y el cursor del ratón.

        Si queremos tener elementos <input readonly> pero formateados como texto plano, podemos usar la clase .form-control-plaintext para eliminar el formato del campo de formulario pero preservar el espaciado.

        Del mismo modo que en los elementos anteriores, las casillas de verificación y botones de opciones, checkbox y radio respectivamente, se controlan mediante la clase .form-check-input. En el caso de llevar etiquetas adjuntas (label) estas llevarían la clase .form-check-label.

        Checkboxes inline y sin etiquetas

        Si deseamos que los checkboxes o radio buttons estén en línea se puede incluir la clase .form-check-inline junto a .form-check. Por otro lado, si no queremos que dispongan de etiqueta podemos omitirla y añadirle la clase .position-static.
