    4. …
    Cambios en el theming de Boostrap

    Una de las novedades de Boostrap 4 es que apuesta por la compilación personalizada, a partir de variables de sus variables SASS/SCSS, frente a la creación de un archivo CSS aparte para la personalización que se empleaba en Bootstrap 3.

    Para cada proyecto, se compilará el código SCSS/SASS y se generarán todos los estilos de la base con el color establecido para la aplicación.

    Inicio rápido

    La manera más rápida de usar Preving Boostrap (o Boostrap) en un proyecto es enlazar los archivos CSS y JS necesarios. En el caso de incluir una versión personalizada de Preving Bootstrap, compilada para la aplicación, no será necesario enlazar Bootstrap.

    CSS

    Incluir la hoja de estilo CSS mediante la etiqueta <link> en el <head> del documento, antes de cualquier otra hoja de estilo.

    JS

    Muchos de los componentes de Bootstrap requieren el uso de JavaScript para funcionar. Concretamente, son necesarios jQuery, Popper.js y los propios complementos de Bootstrap.

    Para ello, hay que incluir los siguientes scripts -en ese orden- al final de la página, justo antes de la etiqueta de cierre </body>, para habilitarlos. jQuery debe aparecer primero, luego Popper.js, y luego nuestros complementos de JavaScript.

    Configuraciones globales y consideraciones finales

    Bootstrap emplea un conjunto de configuraciones globales que es necesario conocer antes de usuarlos; los cuales están casi exclusivamente orientados hacia la normalización de los estilos crossbrowser (para que funcione en el mayor rango de navegadores posible).

    HTML5 doctype

    Es necesario emplear el doctype HTML5. Sin él puede que el aspecto visual se muestre incompleto; incluirlo no debería causar ningún problema considerable.

    Responsive meta tag

    Bootstrap está desarrollado con la concepción mobile first, una estrategia en la que se optimiza primero el código dispositivos móviles y luego se escalan los componentes -según sea necesario- mediante media queries CSS.     Tamaño de caja

    Para trabajar de forma más sencilla, Bootstrap cambiar el valor del tamaño de caja por defecto, content-box a border-box. Esto garantiza que el padding no afecte el ancho final computado de un elemento, pero puede causar problemas con algún software de terceros como Google Maps y Google Custom Search Engine.

    Reset CSS

    Para mejorar la representación en múltiples navegadores, y versiones de los mismos, Bootstrap incorpora un reset CSS. Esto corrige ciertas inconsistencias en los navegadores y dispositivos, a la vez que restablece las propiedades de los elementos mismo tiempo que brindamos restablecimientos de opinión más comunes a elementos HTML comunes.
